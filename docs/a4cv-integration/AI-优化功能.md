  
# AI 优化功能详细说明文档

## 一、功能概述

AI 优化是「A4 可视化简历生成器」中的核心辅助功能，旨在帮助用户将已有简历内容自动整理成结构清晰、表达专业的标准简历格式。该功能通过调用 AI 大语言模型（默认 DeepSeek），在不虚构任何事实的前提下，对简历内容进行结构整理、语言润色和岗位匹配优化。

---

## 二、功能定位与价值

### 2.1 核心价值
| 维度 | 说明 |
|------|------|
| **结构整理** | 将零散文本自动归类到标准简历模块（工作经历、项目经历、教育背景等） |
| **语言优化** | 要点动词开头、量化成果、删除冗余空话 |
| **岗位匹配** | 根据目标 JD 优先呈现相关内容，合理使用岗位关键词 |
| **篇幅控制** | 智能控制输出长度，确保内容能放入 A4 单页 |

### 2.2 适用场景
- 已有 Word/TXT/Markdown 简历，需要快速整理成标准格式
- 简历内容冗长，需要精简优化
- 投递不同岗位，需要针对性调整内容侧重点
- 希望提升简历专业度，但缺乏专业写作经验

```javascript
function estimateCapacity(){
  const base = state.scale.mode==='ratio' ? +state.scale.base : +state.scale.individual.body;
  let c = Math.round(1050 * Math.pow(11 / base, 1.5));
  // 根据布局类型调整容量系数
  if(['typo','mag','intro','card'].includes(layout)) c = Math.round(c * 1.05);
  if(['side-l','side-r'].includes(layout)) c = Math.round(c * 0.95);
  return Math.max(450, Math.min(1600, c));
}
```

**代码位置**：`index.html:4157-4164`

**算法说明**：
- 基准容量：11px 字号下约 1050 字
- 容量与字号呈幂函数关系（1.5 次方）
- 不同布局有不同系数调整（双栏布局略减，杂志布局略增）
- 最终结果限制在 450-1600 字范围

### 4.2 核心算法

#### 篇幅估算算法

```
容量 = 基准容量 × (基准字号 / 当前字号)^1.5 × 布局系数
```

| 布局类型 | 系数 | 说明 |
|----------|------|------|
| 默认布局 | 1.0 | 顶栏、紧凑等标准布局 |
| typo/mag/intro/card | 1.05 | 双栏布局，容量略增 |
| side-l/side-r | 0.95 | 侧栏布局，容量略减 |

**设计意图**：
- 字号越小，可容纳字数越多（幂函数关系比线性更符合实际排版）
- 双栏布局信息密度更高
- 侧栏布局主内容区较小

### 4.3 处理模式对比

| 模式 | 参数值 | 处理策略 | 适用场景 |
|------|--------|----------|----------|
| **基本维持原文** | `keep` | 只做结构整理、归类合并和明显笔误修正，保留原文措辞 | 内容质量较高，只需调整结构 |
| **优化润色内容** | `polish` | 动词开头、量化成果、合并重复、删除冗余空话 | 需要提升专业度，内容较粗糙 |

### 4.4 岗位匹配机制

当用户输入目标岗位 JD 时，系统会：
1. 将 JD 内容截取前 1500 字符（避免超出模型上下文限制）
2. 加入系统提示词，指导 AI：
   - 优先呈现与岗位最相关的内容
   - 合理使用岗位关键词
   - 在不虚构经历的前提下进行优化

### 6.2 运行时错误处理

```javascript
try {
  const resp = await fetch(...);
  if(!resp.ok){
    const t = await resp.text().catch(()=> '');
    throw new Error('接口返回 HTTP ' + resp.status + (t ? '：' + t.slice(0,180) : ''));
  }
  // ... 解析响应
  if(!/^#\s+/m.test(text)) throw new Error('模型返回的不是预期的简历格式，请重试一次。');
} catch(err) {
  const m = (err && err.message) || String(err);
  const hint = /failed to fetch|networkerror|load failed/i.test(m)
    ? '\n可能原因：网络不通 / 接口地址错误 / 浏览器跨域拦截（CORS）。DeepSeek 官方接口支持浏览器直连，请检查网络与 Key。'
    : (/401|invalid|key/i.test(m) 
       ? '\n请检查 API Key 是否正确、是否有余额。' 
       : '');
  aiStatus('处理失败：' + m + hint, true);
}
```

**错误类型识别**：
| 错误特征 | 提示信息 |
|----------|----------|
| 网络错误 | 提示检查网络、接口地址、CORS |
| 401/Key 无效 | 提示检查 API Key 和余额 |
| 格式错误 | 提示模型返回格式不正确 |

## 八、使用建议

### 8.1 最佳实践

1. **准备阶段**：先导入或编辑好简历内容，确保内容完整
2. **选择模式**：
   - 已有优质内容：选择「基本维持原文」
   - 需要提升专业度：选择「优化润色内容」
3. **岗位匹配**：粘贴目标岗位 JD，让 AI 针对性优化
4. **结果校验**：AI 处理完成后，检查是否有不符合预期的修改

### 8.2 常见问题

**Q：为什么 AI 返回内容为空或格式错误？**
- A：可能是 API Key 失效或余额不足；也可能是网络问题或接口地址错误

**Q：处理时间较长（超过 30 秒）？**
- A：AI 处理速度取决于服务商服务器负载，建议耐心等待；如超时可刷新重试

**Q：内容被过度压缩？**
- A：当前版式容量估算可能偏保守，可尝试调大字号或切换更紧凑的布局

**Q：自定义接口无法使用？**
- A：检查接口是否支持 CORS（浏览器直连需要服务端支持跨域）；Ollama 等本地服务需要配置正确的端口和路径
