### 3. 统一 LLM 服务（`app/services/llm.py`）

**核心设计**：一个文件，5 个函数，支持任意 OpenAI 兼容协议（包括智谱）：

```python
"""轻量级 LLM 统一调用层。"""
import json
import logging
import re
from typing import Any

import httpx
from functools import lru_cache

from ..core.config import settings

logger = logging.getLogger(__name__)
FENCE_PATTERN = re.compile(r"```(?:json)?\s*([\s\S]*?)```", re.IGNORECASE)


@lru_cache
def _get_client() -> httpx.AsyncClient:
    return httpx.AsyncClient(
        base_url=settings.LLM_BASE_URL,
        headers={"Authorization": f"Bearer {settings.LLM_API_KEY}"},
        timeout=60.0,
    )


async def chat(prompt: str, system: str = "", **opts) -> str:
    """调用 LLM 聊天接口，返回原始文本。"""
    messages = []
    if system:
        messages.append({"role": "system", "content": system})
    messages.append({"role": "user", "content": prompt})
    
    payload = {
        "model": settings.LL_MODEL,
        "messages": messages,
        "temperature": opts.get("temperature", 0.3),
        "top_p": opts.get("top_p", 0.9),
        **({"max_tokens": opts["max_tokens"]} if "max_tokens" in opts else {}),
    }
    
    resp = await _get_client().post("/chat/completions", json=payload)
    resp.raise_for_status()
    return resp.json()["choices"][0]["message"]["content"]


async def chat_json(prompt: str, system: str = "", **opts) -> dict:
    """调用 LLM 并解析为 JSON（自动处理 ```json``` 代码块）。"""
    text = await chat(prompt, system, **opts)
    return _parse_json(text)


async def embed(text: str) -> list[float]:
    """调用嵌入接口，返回向量。"""
    payload = {"model": settings.EMBEDDING_MODEL, "input": text}
    resp = await _get_client().post("/embeddings", json=payload)
    resp.raise_for_status()
    return resp.json()["data"][0]["embedding"]


def _parse_json(text: str) -> dict:
    """健壮的 JSON 解析：直接 → 代码块 → 第一个 { ... } 块。"""
    text = text.strip()
    # 1) 直接解析
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        pass
    # 2) 提取 ```json``` 代码块
    for m in FENCE_PATTERN.finditer(text):
        try:
            return json.loads(m.group(1).strip())
        except json.JSONDecodeError:
            continue
    # 3) 提取第一个 { ... } 块
    s, e = text.find("{"), text.rfind("}")
    if s != -1 and e > s:
        try:
            return json.loads(text[s:e+1])
        except json.JSONDecodeError:
            pass
    raise ValueError(f"无法从 LLM 输出中解析 JSON: {text[:200]}")
```

## 能不能去掉数据库？

**先回答核心问题:这 4 张表做什么用?**

整个工作流是这样的：
1. 用户上传 PDF → 存到 `resumes` + LLM 提取 → 存到 `processed_resumes`
2. 用户粘贴 JD → 存到 `jobs` + LLM 提取 → 存到 `processed_jobs`
3. 用户点"优化" → 后端通过 `resume_id` + `job_id` 从这 4 张表读出原文和 keywords → 调用 LLM 重写简历 → **直接返回新简历,不存结果**

**也就是说,数据库的作用只有一个：把"上传时的原文 + LLM 提取结果"持久化,供"优化时"读取**。

### 三个去数据库的方案

**方案 A:完全去掉,改用文件存储 + 内存缓存（最简）**
- 原文存本地 `uploads/{uuid}.md` / `uploads/{uuid}.json`
- LLM 提取结果直接写 JSON 文件
- 不支持进程间共享（多 worker 时会找不到 ID 对应的数据）
- 适合单机部署,代码量减少约 30%

**方案 B:用 Python dict 做进程内缓存（最快,但重启丢数据）**
- 用一个全局 `dict[resume_id] = {...}`
- 完全不需要持久化,无文件 I/O
- **致命问题**:重启服务 → 所有 `resume_id` 失效 → 用户要重新上传
- 不推荐,但代码最少

**方案 C:用 SQLite 但只存 2 张表（折中）**
- 删掉 `job_resume` 关联表（死代码）
- `jobs.resume_id` 字段其实也用得不多,可以拆出来
- 整体省不了多少,只是清理死代码

### 我的建议

**走方案 A**,理由：
1. 这是一个**单用户工具**（一次只优化一份简历一个 JD）,不需要关系查询
2. 上传/优化是**短流程**（几分钟内）,没必要走完整 ORM
3. SQLAlchemy + aiosqlite + 4 张表 + 关联关系 ≈ 280 行代码,可全删
4. 改用文件存储后,部署变成"单文件 + 单服务",真正的"轻量化"
