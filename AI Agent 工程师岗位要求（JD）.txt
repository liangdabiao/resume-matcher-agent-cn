AI Agent 工程师岗位要求（JD）
一、 岗位名称

AI Agent 工程师 / LLM Agent 开发工程师 / AI 应用架构师

二、 核心职责

Agent 设计与开发：基于大语言模型（LLM）设计、构建和部署自主AI Agent系统，使其具备规划（Planning）、工具使用（Tool Use）、记忆（Memory）和反思（Reflection）能力。

RAG 系统优化：开发和维护检索增强生成（Retrieval-Augmented Generation）管道，确保Agent能够访问最新的外部知识库。

工具集成：将外部API、数据库、SaaS服务（如Gmail, Notion, GitHub）封装成Agent可调用的工具（Function Calling）。

Prompt 与 推理优化：设计和迭代复杂的提示词（Prompt Engineering），优化模型的推理路径，降低幻觉（Hallucination）和延迟。

评估体系（Eval）建设：搭建自动化测试和评估框架，用于衡量Agent在准确性、毒性、效率和成本方面的表现。

多模态探索（加分项）：探索文本、图像、语音结合的多模态Agent应用场景。

三、 硬性技能要求 (Hard Skills)
1. 编程与工程基础 (必备)

Python: 精通Python编程，熟悉异步编程（Asyncio）。

后端开发: 熟悉 FastAPI / Flask / Django 等Web框架，能够构建高并发的API服务。

数据工程: 熟悉 SQL/NoSQL 数据库，了解向量数据库（Milvus, Pinecone, Qdrant, ChromaDB）。

DevOps: 熟悉 Docker, Kubernetes, CI/CD 流程，有云原生（AWS/Azure/GCP）部署经验。

2. 大模型生态与框架 (核心)

LLM 框架: 深入理解并熟练使用 LangChain, LlamaIndex, Semantic Kernel, CrewAI​ 或 AutoGen​ 等主流Agent开发框架。

模型 API: 熟悉 OpenAI, Anthropic (Claude), Google (Gemini) 以及开源模型（Llama 3, Qwen, DeepSeek）的API调用与微调。

Prompt 工程: 掌握 Few-shot, Chain-of-Thought (CoT), ReAct, Reflection 等提示词技巧。

3. Agent 核心技术栈

RAG (检索增强生成): 熟悉文档切分（Chunking）、Embedding模型选型、混合检索（Hybrid Search）、重排序（Reranking）等技术。

Function/Tool Calling: 能够将复杂业务逻辑封装为Agent可调用的JSON Schema工具。

Memory 机制: 了解短期记忆（Context Window管理）和长期记忆（Vector Store + 摘要）的实现方案。

Agent 架构: 熟悉 ReAct, Plan-and-Execute, 多Agent协作（Multi-Agent Systems）等架构模式。

4. 评估与监控 (Critical)

熟悉 LangSmith, Weights & Biases (W&B), Arize AI​ 等LLMOps工具。

能够定义评估指标（如 Faithfulness, Answer Relevance, Tool Selection Accuracy）。

四、 软技能与综合素质

第一性原理思维：能够从底层逻辑拆解复杂问题，不局限于现有工具的封装。

快速学习能力：LLM领域技术迭代极快，需要每周跟进最新的论文（ArXiv）、开源项目和博客。

产品思维：不仅关注技术实现，更关注用户体验和最终的业务价值（降本增效）。

调试能力：具备极强的耐心和逻辑推理能力，能够调试非确定性（Non-deterministic）系统的错误。

五、 加分项 (Nice to Have)

开源贡献：在 GitHub 上有高质量的 LLM 相关项目，或为知名框架贡献过代码。

学术背景：发表过 NLP/AI 相关顶会论文（ACL, EMNLP, NeurIPS）。

特定领域知识：金融、法律、医疗或代码生成（Code Interpreter）领域的专业知识。

前端能力：熟悉 React/Vue，能够开发 Agent 的交互界面（如 Chatbot UI）。

模型微调：有 Lora, QLora, SFT 等模型微调实战经验，能针对特定任务优化小模型。
